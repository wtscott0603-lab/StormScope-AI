# Support

## Getting Help

- Use the bug report template for reproducible defects.
- Use the feature request template for product or UX ideas.
- For local setup questions, start with [README.md](README.md) and [SETUP.md](SETUP.md).
- General public support for this project runs through GitHub Issues.

## What To Include

When asking for help, include:

- Your operating system and Docker version
- The exact command you ran
- Relevant logs or error output
- Whether you are using a local `.env`

## Security

Do not use public support channels for vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Maintainer Contact

If an issue is sensitive and should not be discussed in public, do not open a normal support issue. Follow [SECURITY.md](SECURITY.md) instead.
